# 调研SOP标准与质量门禁

> 所有内容产出必须遵循本文档定义的标准。
> 不通过质量门禁的内容不得发布。

---

## 一、信息采集SOP

### 1.1 自动化采集（每日06:00执行）

```
输入：关键词列表 + RSS源列表
处理：
  1. 从Google News RSS拉取最新文章（关键词：AI副业/AI赚钱/AI side hustle/AI monetization）
  2. 从GitHub Trending获取AI项目日增star
  3. 从Product Hunt获取AI产品发布
  4. 按关键词过滤，去重
输出：原始信号池（JSON格式）
```

**信号JSON格式：**
```json
{
  "id": "signal-YYYYMMDD-NNN",
  "date": "YYYY-MM-DD",
  "title": "标题",
  "source": "来源名称",
  "url": "原文链接",
  "summary": "一句话摘要",
  "keywords": ["关键词1", "关键词2"],
  "relevance": "ai-side-hustle|ai-tool|ai-trend|ai-regulation",
  "status": "raw",
  "credibility": null,
  "verifiedAt": null
}
```

### 1.2 信号筛选（每日09:00执行）

**筛选条件（必须全部满足）：**
1. 与AI赚钱/副业/创业直接相关
2. 有可追溯的原始链接
3. 非纯广告/软文（见软文识别规则）
4. 近7天内发布

**软文识别规则（满足≥2项即判定为软文）：**
- 标题含"月入XX万""年赚XX万"等具体数字但正文无数据来源
- 文末有课程/社群/工具的付费引导
- 来源为知识付费平台/个人公众号（非媒体）
- 正文大量使用感叹号和情绪化表达
- 无法找到其他独立来源佐证

### 1.3 快速验证（每日12:00执行，每条≤30分钟）

```
Step 1: 来源验证（5min）
├── 原始链接是否可访问
├── 发布日期是否合理
├── 作者/媒体是否有公信力
└── 输出：来源可信度（高/中/低）

Step 2: 反面搜索（10min）
├── 搜索"[关键词] + 骗局/套路/投诉"
├── 搜索"[关键词] + 辟谣/假的/不实"
├── 检查中消协/黑猫投诉是否有相关投诉
└── 输出：是否有反面证据（有/无）

Step 3: 交叉验证（10min）
├── 搜索同一事件的其他报道
├── 至少需要2个独立来源
├── 对比不同来源的数据是否一致
└── 输出：交叉验证结果（一致/矛盾/无其他来源）

Step 4: 初步评分（5min）
├── 可信度评分：1-5星
├── 是否值得深度调研：是/否
├── 紧急程度：高/中/低
└── 输出：验证报告（存入信号JSON）
```

## 二、深度调研SOP

### 2.1 调研启动标准

**进入深度调研的条件（必须全部满足）：**
- 快速验证可信度≥3星
- 至少有2个独立来源
- 无被删除/否认/辟谣的记录
- 与AI赚钱/副业直接相关

### 2.2 调研流程

```
Phase 1: 市场需求验证（30min）
├── 搜索量数据（百度指数/Google Trends）
├── 平台数据（闲鱼在售数/Fiverr gig数/小红书笔记数）
├── 竞品分析（已有多少人在做）
├── 用户需求信号（论坛讨论/评论区反馈）
└── 输出：需求验证结论（强/中/弱）

Phase 2: 收入数据验证（30min）
├── 收集多个来源的收入数据
├── 对比不同来源的数据差异
├── 计算收入区间（P25-P75，不取极端值）
├── 标注数据来源和可信度
└── 输出：收入区间估算表

Phase 3: 工具链梳理（30min）
├── 列出所需全部工具
├── 区分免费/付费方案
├── 计算最低启动成本
├── 标注每个工具的学习曲线
└── 输出：工具链清单

Phase 4: 法律风险评估（20min）
├── 检查5项红线：
│   ├── [ ] 是否涉及数据抓取（反不正当竞争法§2/§12）
│   ├── [ ] 是否涉及用户隐私（个人信息保护法）
│   ├── [ ] 是否涉及版权/知识产权
│   ├── [ ] 是否涉及金融/投资建议（证券法）
│   └── [ ] 是否涉及虚假宣传（广告法）
├── 搜索相关判例
└── 输出：法律风险等级（低/中/高/红线）

Phase 5: SOP拆解（30min）
├── 前提条件清单
├── 第一天就能做的事
├── 7天MVP路径
├── 时间投入 vs 收入预期量化模型
└── 输出：执行SOP

Phase 6: 文档撰写（30min）
├── 按模板格式撰写
├── 所有结论附来源
├── 通过质量门禁自检
└── 输出：完整调研文档
```

### 2.3 收入数据估算规范

**禁止写法：**
- ❌ "月入10万"（无区间、无来源）
- ❌ "年赚200万"（极端值、营销话术）
- ❌ "保证赚钱"（违规表述）

**标准写法：**
- ✅ "月收入区间：¥2,000-8,000（P25-P75，基于闲鱼Top20卖家数据）"
- ✅ "时薪估算：¥30-80/小时（基于Fiverr AI writing类别Top50 gig均价）"
- ✅ "起步阶段（第1-2月）：¥0-2,000/月；成长阶段（第3-6月）：¥2,000-8,000/月"

## 三、质量门禁清单

### 3.1 发布前强制检查（10项全部通过才能发布）

```
信息质量（4项）
□ 1. 每个结论有至少1个可追溯来源（附链接+日期）
□ 2. 可信度评分已填写（附评分依据）
□ 3. 收入数据有区间估算（P25-P75，不写单一数字）
□ 4. 已搜索反面证据（负面报道、用户投诉、辟谣）

执行可行性（3项）
□ 5. 执行SOP可在7天内启动（不依赖复杂前置条件）
□ 6. 工具链有免费方案（不强制付费才能开始）
□ 7. 第一天就能做的事已明确列出

合规性（3项）
□ 8. 法律风险已评估（5项红线检查完成）
□ 9. 无"保证赚钱""月入XX万"等违规表述
□ 10. 已标注信息状态（待验证/验证中/已验证/已证伪）
```

### 3.2 质量等级

| 等级 | 标准 | 说明 |
|------|------|------|
| **A级** | 10/10项通过 + 可信度≥4星 | 可直接发布 |
| **B级** | 8/10项通过 + 可信度≥3星 | 可发布但需标注待补充项 |
| **C级** | 6/10项通过 | 仅存入内部研究池，不对外发布 |
| **D级** | <6项通过 | 退回重新调研或标记为已证伪 |

## 四、证伪标准

### 4.1 必须标记为"已证伪"的情况

1. **来源被删除**：原始文章/视频被发布者删除且无备份
2. **官方否认**：平台/机构官方已发布否认/辟谣声明
3. **数据矛盾**：多个独立来源给出严重矛盾的数据，无法调和
4. **无法复现**：实际操作后无法达到描述的效果
5. **监管通报**：被中消协/市场监管总局/法院认定为虚假/违规
6. **法律风险**：深度调研后发现存在明确法律红线

### 4.2 证伪记录格式

```markdown
## 证伪记录

**方向：** [方向名称]
**证伪日期：** YYYY-MM-DD
**原可信度：** ⭐⭐⭐
**证伪原因：** [具体原因]
**证据：**
1. [证据1+链接]
2. [证据2+链接]
**教训：** [下次如何避免类似误判]
```
