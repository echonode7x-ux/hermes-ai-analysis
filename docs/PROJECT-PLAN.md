# 项目计划书

> AI Venture Lab — 可执行商业情报平台
> 版本：v1.0 | 日期：2026-05-25

---

## 一、项目总览

### 1.1 项目目标

建立一套**可持续运行的AI创业情报系统**，实现：
- 每日自动采集AI行业信号
- 每周产出2-3篇深度调研（含完整SOP）
- 每月更新全景图和趋势预判
- 所有内容可追溯、可验证、可执行

### 1.2 成功标准（可量化）

| 指标 | 3个月目标 | 6个月目标 | 12个月目标 |
|------|-----------|-----------|------------|
| 深度调研文档数 | 30篇 | 80篇 | 200篇 |
| 已验证方向数 | 15个 | 40个 | 100个 |
| 预测准确率 | 60% | 70% | 80% |
| 自动化覆盖率 | 50% | 70% | 85% |
| 站点日均UV | 50 | 200 | 1000 |

## 二、模块计划

### 模块1：AI副业全景图

**目标：** 建立2026年AI副业方向的完整数据库

**交付物：**
- `content/research/landscape.md` — 全景图文档
- `site/data/landscape.json` — 结构化数据
- 站点页面：opportunities.html 增加副业分类视图

**人财物需求：**
| 资源 | 需求 | 来源 | 成本 |
|------|------|------|------|
| 人力 | 调研分析师×1 | AI+人工 | 每日2h |
| 数据源 | Google News RSS、即刻、V2EX、Reddit | 免费API | 0元 |
| 工具 | Hermes Agent + MiMo | 已有 | 0元 |
| 存储 | Git仓库 | 已有 | 0元 |

**执行计划：**
```
第1天：信号采集 — 搜集50+个AI副业方向的原始信号
第2天：分类整理 — 按"零门槛/有门槛/高门槛"三级分类
第3天：快速验证 — 每个方向做30分钟初步验证
第4天：深度调研 — 选Top10方向做完整调研
第5天：全景图输出 — 文档+JSON+站点更新
第6天：质量自检 — 通过质量门禁清单
第7天：Git提交+部署
```

**验证标准：**
- [ ] 覆盖≥15个AI副业方向
- [ ] 每个方向有可信度评分
- [ ] 每个方向有法律风险标记
- [ ] JSON数据结构完整，站点可渲染
- [ ] 通过质量门禁清单全部10项

### 模块2：可行性验证体系

**目标：** 对每个方向做"正反攻击"式验证

**交付物：**
- 验证报告模板
- 10个方向的完整验证报告
- 证伪案例库（记录已证伪的方向和原因）

**人财物需求：**
| 资源 | 需求 | 来源 | 成本 |
|------|------|------|------|
| 人力 | 调研分析师×1 + 合规审查×1 | AI+人工 | 每日3h |
| 数据源 | 中消协公告、裁判文书网、平台政策 | 免费 | 0元 |
| 工具 | Hermes Agent + 搜索引擎 | 已有 | 0元 |

**执行计划：**
```
第1周：建立验证框架
├── 定义"正反攻击"标准流程
├── 建立法律红线检查清单
├── 创建验证报告模板
└── 输出：SOP文档

第2-3周：批量验证
├── 对全景图中Top10方向逐个验证
├── 每个方向：正面证据收集 + 反面证据搜索 + 法律风险评估
├── 输出：10篇验证报告
└── 至少3个方向标记为"已证伪"

第4周：体系优化
├── 根据验证结果优化SOP
├── 建立证伪案例库
└── 输出：优化后的验证框架
```

**验证标准：**
- [ ] ≥10个方向完成完整验证
- [ ] 每篇报告通过质量门禁
- [ ] ≥3个方向被证伪（附证伪原因）
- [ ] 验证流程可复用（新方向可用同一SOP）

### 模块3：SOP拆解库

**目标：** 为每个已验证方向产出可执行的SOP

**交付物：**
- `content/templates/` — SOP模板库
- 每个已验证方向的独立SOP文档
- 工具链清单（免费/付费对比）

**人财物需求：**
| 资源 | 需求 | 来源 | 成本 |
|------|------|------|------|
| 人力 | 调研分析师×1 | AI+人工 | 每日2h |
| 工具 | 各平台账号（用于实测） | 注册 | 0元 |
| 测试 | 小规模实操验证 | 人工 | 每方向2-4h |

**执行计划：**
```
第1周：SOP模板设计
├── 统一SOP格式（前提条件→工具链→7天MVP→收入预期）
├── 设计时间投入 vs 收入预期的量化模型
└── 输出：SOP模板

第2-4周：逐方向拆解
├── 每个已验证方向产出独立SOP
├── 每个SOP必须包含"第一天就能做的事"
├── 工具链标注免费方案
└── 输出：10个方向的SOP

持续：SOP迭代
├── 根据实操反馈更新SOP
├── 淘汰无效步骤
├── 新增发现的捷径
```

**验证标准：**
- [ ] 每个SOP可在7天内启动（不依赖前置条件）
- [ ] 每个SOP有免费工具方案
- [ ] 每个SOP有量化的时间/收入预期
- [ ] 每个SOP有"第一天就能做的事"

### 模块5：AI短剧/漫剧深度调研

**目标：** 验证"一人公司"AI短剧模式的可行性

**交付物：**

**人财物需求：**
| 资源 | 需求 | 来源 | 成本 |
|------|------|------|------|
| 人力 | 调研分析师×1 | AI+人工 | 每日3h×5天 |

**执行计划：**
```
第1天：工具实测
├── 注册可灵AI、即梦AI、Pixverse
├── 测试各工具的免费额度、生成质量、操作复杂度
└── 输出：工具对比表

├── 搜索AI短剧相关报道（北京商报、潮新闻等）

第3天：变现路径分析
├── 平台分成机制（抖音/快手/视频号）
├── 定制接单市场（闲鱼/小红书）

└── 输出：变现路径图

第4天：法律风险评估
├── AI短剧《霍去病》争议分析
├── AI生成内容版权问题
├── 各平台对AI内容的政策
└── 输出：法律风险报告

第5天：SOP输出
├── 完整执行SOP
├── 工具链推荐
├── 成本/收入预期
└── 输出：完整调研文档
```

**验证标准：**
- [ ] 工具实测有截图/数据
- [ ] 收入数据有多个来源交叉验证
