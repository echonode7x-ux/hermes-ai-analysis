# 运行模型（Operating Model）

> 本文档定义 AI Venture Lab 的核心运行方法论。
> 所有内容产出必须遵循本模型，确保信息真实、可执行、可进化。

---

## 一、信息真实性模型

### 1.1 信息分级体系

| 级别 | 定义 | 来源要求 | 示例 |
|------|------|----------|------|
| **L1 一手验证** | 亲自操作、注册、测试 | 本人实际操作截图/录屏 | 注册Fiverr，发布一单，记录真实反馈 |
| **L2 权威引用** | 来自可信媒体/机构 | 附原文链接 + 发布日期 | 路透社报道企业裁员转向AI投资 |
| **L3 平台数据** | 可公开查询的平台数据 | 附查询路径 + 时间戳 | 闲鱼搜索"AI代写"显示在售商品数 |
| **L4 社交传播** | 社交媒体/论坛讨论 | 附原始链接 + 交叉验证 | Reddit用户分享收入截图 |
| **L5 未验证** | 猜测/推断/听说 | 明确标注"未验证" | "据说有人靠这个月入10万" |

### 1.2 可信度评分（1-5星）

```
⭐      L5未验证，单一来源，无法追溯
⭐⭐    L4社交传播，单一来源，未交叉验证
⭐⭐⭐  L3平台数据 或 L2权威引用，单一来源
⭐⭐⭐⭐ L2权威引用 + L3平台数据交叉验证
⭐⭐⭐⭐⭐ L1一手验证 + L2/L3交叉验证
```

### 1.3 信息采集红线

**以下信息直接丢弃，不进入研究池：**
- ❌ 无原始链接的"暴富案例"
- ❌ 来自知识付费课程的"学员收入展示"
- ❌ 微信群/朋友圈的截图（无法验证）
- ❌ 已被官方删除/否认的内容（如"程前朋友圈"删帖事件）
- ❌ 标题党但正文无实质数据的文章

## 二、调研SOP标准

### 2.1 单条调研SOP（标准流程）

```
Phase 1: 信号捕获（15分钟）
├── 来源：Google News RSS / 即刻 / V2EX / Reddit / Product Hunt
├── 筛选条件：与AI赚钱/副业/创业相关
├── 输出：原始链接 + 标题 + 一句话摘要
└── 门槛：至少2个独立来源提及同一方向

Phase 2: 快速验证（30分钟）
├── 搜索原始报道，确认不是营销软文
├── 检查是否被删除/否认/辟谣
├── 查找反面证据（负面报道、用户投诉）
├── 初步可信度评分
└── 输出：验证报告（500字以内）

Phase 3: 深度调研（2-4小时）
├── 市场需求验证：搜索量、平台数据、竞品数量
├── 收入数据验证：多来源交叉对比
├── 工具链梳理：免费/付费方案对比
├── 法律风险评估：红线检查清单
├── 执行SOP拆解：从0到第一笔收入的最小路径
└── 输出：完整调研文档（2000-5000字）

Phase 4: 质量门禁（自检清单）
├── [ ] 每个结论有至少1个可追溯来源
├── [ ] 收入数据有区间估算（不写单一数字）
├── [ ] 法律风险已评估（附红线清单）
├── [ ] 执行SOP可在7天内启动
├── [ ] 工具链有免费方案（不强制付费）
├── [ ] 已标注信息级别和可信度
└── [ ] 已检查是否涉及数据抓取/隐私/版权

Phase 5: 发布与追踪
├── Git commit（Conventional Commits格式）
├── 更新 site/data/ JSON
├── 标记状态：待验证 → 验证中 → 已验证 → 已证伪
└── 设定30天后复查提醒
```

## 三、每日执行循环

### 3.1 日程表

| 时间 | 任务 | 输入 | 输出 | 耗时 |
|------|------|------|------|------|
| 06:00 | 信号采集 | RSS/API/搜索 | 原始信号池 | 30min |
| 09:00 | 信号筛选 | 原始信号池 | 筛选后信号（≤5条） | 15min |
| 12:00 | 快速验证 | 筛选后信号 | 验证报告 | 30min |
| 18:00 | 内容产出 | 验证报告 | 文档/信号/研究 | 1-2h |
| 21:00 | 数据更新 | 产出内容 | JSON+Git | 15min |

### 3.2 信号采集源

**自动化源（定时任务）：**
- Google News RSS：关键词 `AI副业` `AI赚钱` `AI side hustle` `AI monetization`
- GitHub Trending：AI相关项目日增star
- Product Hunt：AI产品发布

**半自动源（人工辅助）：**
- 即刻：AI相关话题
- V2EX：AI/Creative节点
- Reddit：r/artificial, r/SideProject, r/Entrepreneur
- 小红书：AI副业相关笔记（搜索验证）

**人工源（定期检查）：**
- 中消协/市场监管总局公告
- 各平台政策更新（抖音/微信/闲鱼/小红书）

### 3.3 每日最低产出标准

| 日类型 | 最低产出 | 说明 |
|--------|----------|------|
| 工作日 | 1条每日信号 + 更新JSON | 保持节奏 |
| 研究日 | 1篇深度调研文档 | 每周至少2天 |
| 休息日 | 无强制要求 | 但信号采集不停 |

## 四、进化机制

### 4.1 状态流转

```
新发现 → 待验证 → 验证中 → 已验证 → 已证伪
   ↑                                      |
   └──────── 30天后复查 ←─────────────────┘
```

**每个调研条目必须有明确状态：**
- **待验证**：刚发现，尚未做深度调研
- **验证中**：正在做Phase 2-3
- **已验证**：通过质量门禁，可信度≥3星
- **已证伪**：验证后发现不实/不可行，附证伪原因
- **已过期**：超过30天未更新，标记为需复查

### 4.2 周复盘框架

每周日执行，输出到 `content/daily/YYYY-MM-DD-review.md`：

```markdown
# 周复盘 YYYY-WXX

## 本周产出
- 信号：X条
- 深度调研：X篇
- SOP：X个

## 验证结果
| 方向 | 原判断 | 实际结果 | 差异原因 |
|------|--------|----------|----------|

## 预测准确率
- 本周预测：X条
- 验证正确：X条
- 准确率：XX%

## 下周调整
- 新增方向：
- 淘汰方向：
- 优先级调整：
```

### 4.3 月度全景更新

每月1号执行，更新 `content/research/landscape.md`：

1. 刷新所有方向的市场数据
2. 更新收入区间估算
3. 淘汰连续2个月无进展的方向
4. 新增本月发现的新方向
5. 更新法律环境变化

### 4.4 季度审计

每季度执行一次全面审计：

1. **数据刷新**：所有引用链接是否存活，数据是否过时
2. **收入验证**：对比预测收入 vs 已知真实案例
3. **方法论迭代**：SOP是否有优化空间
4. **方向调整**：根据6个月数据，调整优先级

## 五、质量门禁清单

### 5.1 发布前检查（每篇内容必须通过）

```
□ 信息来源标注完整（链接+日期+级别）
□ 可信度评分已填写（附评分依据）
□ 收入数据有区间估算（不写单一数字，不写"月入XX万"）
□ 法律风险已评估（5项红线检查）
□ 执行SOP可在7天内启动
□ 工具链有免费方案（不强制付费才能开始）
□ 已检查是否涉及数据抓取/隐私/版权
□ 已搜索反面证据（负面报道、用户投诉、辟谣）
□ 已标注信息状态（待验证/验证中/已验证/已证伪）
□ 无"保证赚钱"等违规表述
```

### 5.2 证伪标准

以下情况必须标记为"已证伪"：
- 原始来源已删除且无备份
- 平台官方已否认/回应
- 多个独立来源给出矛盾数据
- 实际操作后无法复现
- 涉及法律风险被监管部门通报

## 六、人财物需求模型

### 6.1 角色定义

本项目为"一人公司"运营，但需要以下角色能力：

| 角色 | 职责 | 每日投入 | AI辅助率 |
|------|------|----------|----------|
| 信息采集员 | 信号搜集、RSS监控 | 30min | 80%（AI自动采集+筛选） |
| 调研分析师 | 深度调研、验证 | 2h | 60%（AI辅助搜索+整理） |
| 内容编辑 | 文档撰写、格式化 | 1h | 70%（AI初稿+人工精修） |
| 产品经理 | 方向判断、优先级 | 30min | 30%（主要靠经验和数据） |
| 合规审查员 | 法律风险评估 | 15min | 50%（AI辅助检索+人工判断） |

**总计：每日4-5小时，AI辅助率约65%**

### 6.2 工具清单

| 工具 | 用途 | 成本 | 必要性 |
|------|------|------|--------|
| Hermes Agent | 自动化采集+分析+产出 | 已有 | 必须 |
| Google News RSS | 新闻信号源 | 免费 | 必须 |
| GitHub API | 项目/趋势监控 | 免费 | 必须 |
| DeepSeek/MiMo | 内容生成+分析 | 已有 | 必须 |
| Cloudflare Pages | 部署 | 免费 | 必须 |
| Git/GitHub | 版本管理 | 免费 | 必须 |
| 各平台账号 | 验证+数据采集 | 免费 | 必须 |

**月度运营成本：≈0元（全部使用免费/已有工具）**

### 6.3 时间投入模型

```
启动期（第1-2周）：每日6-8小时
├── 搭建自动化管线
├── 完成首批10个方向的深度调研
├── 建立信号采集源
└── 产出：10篇调研文档 + 自动化脚本

稳定期（第3周起）：每日4-5小时
├── 信号采集：30min（80%自动化）
├── 快速验证：30min
├── 深度调研：2h（每周2-3篇）
├── 内容产出：1h
└── 产出：每周2-3篇深度调研 + 每日信号

维护期（第3个月起）：每日2-3小时
├── 自动化管线成熟
├── 主要做验证、更新、复盘
└── 产出：每周1-2篇 + 月度全景更新
```
