## 一、产品定义

### 产品名称
**PriceGuard（价格卫士）** — AI驱动的竞品价格监控服务

### 核心功能

| 功能 | 说明 |
|------|------|
| 竞品价格采集 | 用户输入商品URL，系统自动提取当前价格、标题、主图、SKU信息 |
| 定时监控 | 用户设置监控频率（1h/4h/12h/24h），系统自动定时采集 |
| 价格变化告警 | 价格变动超过阈值（默认±5%）时，通过微信/短信/邮件通知 |
| AI分析报告 | 基于价格趋势数据，AI生成竞品定价策略分析与建议 |
| 数据可视化 | 价格走势图、竞品对比表、历史数据导出（CSV/Excel） |
| 多平台支持 | 淘宝、天猫、京东、拼多多、1688、抖音电商 |

### 用户场景

**场景1：新品定价**
> 卖家小王准备上架一款蓝牙耳机，在PriceGuard输入Top 20竞品链接，系统返回价格分布区间（¥39-¥129），AI建议定价¥69（中位价偏下），并指出¥59-¥79是竞争最激烈的价格带。

**场景2：促销跟踪**
> 卖家小李监控5个核心竞品，某竞品突然降价15%，PriceGuard 5分钟内推送告警，小李决定是否跟进降价或调整推广策略。

**场景3：周期性报告**
> 每周一早上9点，PriceGuard自动推送上周竞品价格周报：价格变化汇总、异常波动标记、定价建议。

### 差异化优势

| 对比维度 | 传统做法 | PriceGuard |
|----------|----------|------------|
| 数据获取 | 人工逐个查看，每天30分钟 | 自动采集，5秒/商品 |
| 价格变化 | 发现时已错过应对窗口 | 实时告警，最快5分钟响应 |
| 分析能力 | 凭经验判断 | AI量化分析，给出具体建议 |
| 合规性 | 灰色地带爬虫 | 用户授权触发，不建公共库 |
| 使用门槛 | 需要技术团队 | 输入链接即可，零技术门槛 |

### 核心模块

#### 模块1：智能采集引擎

```python
# 核心采集逻辑伪代码
class Collector:
    async def collect(self, url: str) -> ProductInfo:
        platform = self.detect_platform(url)  # 淘宝/京东/拼多多...
        strategy = self.get_strategy(platform)  # 平台特定策略
        
        if strategy.needs_browser:
            # 动态页面用Playwright
            data = await self.browser_fetch(url, strategy)
        else:
            # 静态/API页面用httpx
            data = await self.http_fetch(url, strategy)
        
        return self.parse(data, strategy)
```

#### 模块2：AI分析引擎

```
输入：商品价格历史数据 + 竞品价格数据
```

## 四、法律合规

### 合规方式

**核心原则：用户授权 + 不建公共库 + 不绕过反爬**

| 合规要点 | 具体做法 |
|----------|----------|
| 用户主动触发 | 用户主动输入URL，系统代理采集，属于"代替用户浏览" |
| 不建公共数据库 | 数据仅存储在用户个人账户下，不共享、不聚合、不售卖 |
| 不绕过反爬机制 | 不破解验证码、不模拟登录、不使用黑产IP池 |
| 遵守robots.txt | 采集前检查目标站robots.txt规则 |
| 合理频率 | 单商品最低间隔30分钟，不超过正常浏览频率 |
| 数据最小化 | 仅采集价格、标题、主图URL，不采集用户评论等UGC内容 |
| 用户协议 | 明确告知用户数据用途，用户可随时删除 |

### 风险点

| 风险 | 严重程度 | 概率 | 说明 |
|------|----------|------|------|
| 平台起诉（反不正当竞争） | 高 | 低 | 参考大众点评vs百度案，但场景不同 |
| IP被封/验证码 | 中 | 高 | 日常运营问题，需技术应对 |
| 平台改版导致采集失败 | 中 | 高 | 需持续维护适配 |
| 用户协议违规指控 | 中 | 中 | 需法务审查平台ToS |
| 数据安全合规 | 低 | 低 | 需符合《个人信息保护法》 |

### 应对措施

**法务层面：**
1. 开发前请律师审查产品模式，预算¥5,000-10,000
2. 用户协议中明确"用户授权代为访问"条款
3. 不在营销中使用"爬虫""破解"等敏感词，用"智能采集""数据代理"
4. 保留完整的用户授权日志

**技术层面：**
1. 使用正规代理IP服务（如快代理、芝麻代理），¥500-2,000/月
2. 采集频率严格控制，单IP对单域名≤2请求/分钟
3. 实现robots.txt自动解析和遵守
4. 遇到验证码自动跳过，不尝试破解
5. 采集结果缓存，减少重复请求

**运营层面：**
1. 用户协议明确禁止用于恶意竞争目的
2. 设置合理的使用上限，防止滥用
3. 如收到平台律师函，立即暂停相关平台采集功能
4. 购买商业保险（互联网责任险），年费约¥3,000-8,000

**合规参考案例：**
- 慢慢买、什么值得买：公开比价，运营多年未被起诉
- Keepa、CamelCamelCamel：亚马逊价格追踪，全球运营
- 关键区别：我们是用户授权+个人数据，不是公开数据库
